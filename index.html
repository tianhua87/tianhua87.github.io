原来浏览器默认是不允许在 https 里面引用 http 资源的，比如引入一个 js 文件，会被直接 block 掉的。

#解决方法

1 相对协议

去掉url前面的协议名”http:”或”https”,让浏览器自动去补齐，如：

2 使用 iframe

使用 iframe 的方式引入 http 资源，比如在 https 里面播放优酷的视频，我们可以先在一个 http 的页面里播放优酷视频，然后将这个页面嵌入到 https 页面里就可以了。

另外一个典型的例子是在 https 页面里通过 Ajax 的方式请求 http 资源，Chrome 是不允许直接 Ajax 请求 http 的。如果两个页面的内容都可以控制的话，当前窗口可以 iframe 窗口进行通信。

  • 适配器的代码如下，

2 经过日志的打印输出，发现onCreateViewHolder(ViewGroup parent, int viewType)、onBindViewHolder(ViewHolder holder, int position)、getItemCount()这些函数都正常运行了一遍，然后程序就奔溃了，我的测试数据alarmList里面是有5组数据的，结果只打印了1次出来，程序就奔溃了，好难排查，主要是我的华为G9 Plus有一些开关没开，有很多错误信息没有显示出来，很难排查错误。

3 解决方法

最后还是舍友发现了一个不一样的地方，View view= LayoutInflater.from(context).inflate(resourceID,parent);，inflate()参数只有resourceID和parent，没有boolean attachToRoot这个函数，一般都是View view= LayoutInflater.from(context).inflate(resourceID,parent,false);,果然，加上false之后程序就正常了。

4 原理

原理参考CSDN的一篇文章：http://m.blog.csdn.net/qq_33165549/article/details/71555837

inflate()的两种参数

5 来看看源码

  • inflate(int resource, ViewGroup root)、inflate(int resource, ViewGroup root, boolean attachToRoot)

实际上inflate(int resource, ViewGroup root)还是调用了inflate(int resource, ViewGroup root, boolean attachToRoot)，也就是说当我只传入root时，就相当于调用了inflate(resource, root, true),但是为什么会奔溃呢？我依然没有理解。
解释：如果resource对应的那个控件本身在他所在的布局文件里面没有父控件的话，attachToRoot可以为true，如果有父控件的话，就只能设置为false,不然的话就会报错误java.lang.IllegalStateException: The specified child already has a parent. You must call removeView() on the child’s parent first

RadioGroup调用check()方法时onCheckedChanged()被多次调用多次

1使用ViewPager进行左右滑动时有明显卡顿

1 页面布局中间是一个ViewPager，底部是导航栏


开始以为是Fragment频繁被销毁和创建，倒是加载太耗时，但是解决了Fragment的销毁和创建后依然滑动卡顿，后来觉得可能是RadioButton换背景图片太耗时,但是把所有换背景的代码注释掉后依然没有作用。然后又把chexk()方法注释掉了，发现变流畅了，最后在onCheckedChanged()方法中打印了一条日志，发现滑动一次时onCheckedChanged方法被调用了3次之多

2 那就来看看这个check()的源码

  • check()

发现调用了setCheckedStateForView(mCheckedId, false) 和setCheckedId(id)这两个函数

  • 来看看setCheckedId(int id)

最终调用了mOnCheckedChangeListener.onCheckedChanged(this, mCheckedId)这个函数,所以这里就调用了1次。

  • setCheckedStateForView(int mCheckedId, boolean checked)，内部又调用了RadioButton的setChecked(boolean checked)

  • 再来看setChecked(checked)

这里确实有调用onCheckedChanged()方法，可是，我们没有给RadioButton调用setOnCheckedChangeListener()方法啊，所以mOnCheckedChangeListener==null，这是怎么回事，其余的两次哪里调用的。如果有调用只能是mOnCheckedChangeWidgetListener.onCheckedChanged()里调用了。

原来RadioGroup在添加Child的时候会给Child设置CompoundButton.OnCheckedChangeListener

mChildOnCheckedChangeListener是CheckedStateTracker类的实例，CheckedStateTracker实现

我们看到再次调用了setCheckedId(id)这个方法，这也就是为什么onCheckedChanged被多次调用的原因了
